## Database

Two Supabase tables:

- **products**: id, name, slug, description, price_clp (int), compare_at_clp (int?), category_id (FK), image_url, images (text[]), stock, is_active, is_featured, created_at, updated_at
- **categories**: id, name, slug, sort_order

## Auth

- Admin users identified by `user_metadata.is_admin === true`
- Middleware (`proxy.ts`) redirects `/admin/*` (except `/admin/login`) to login if unauthenticated
- Login at `/admin/login` — email+password via Supabase, checks `is_admin`, signs out if not admin
- Slug generated by `slugify(name) + random 4-char suffix` in Server Action

## Image upload

- Supabase storage bucket: `product-images`
- File path: `products/{uuid}.{ext}`
- Upload component in `image-upload.tsx` — hidden input stores the public URL

## Conventions

- **Spanish** for UI text (admin panel, store), English in hero mock content
- **CLP** prices formatted via `formatCLP()` utility
- **OKLCH** color space in `globals.css` — warm cream/taupe palette, light-only (`.dark` duplicates light values)
- **shadcn base-nova** style — `@theme inline` tokens in CSS, `cn()` utility
- **Server Components** by default; "use client" only for interactivity (forms, header)
- Cart/favorites are UI-only — no backend cart implementation exists
- `pnpm-lock.yaml` present — always use `pnpm`, never `npm` or `yarn`
